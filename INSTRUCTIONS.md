# PsychoTeleBot — Инструкция по возможностям бота

## Общее описание

PsychoTeleBot — Telegram-бот для психологической поддержки с интеграцией ИИ, системой заявок и ролевым доступом. Бот позволяет пользователям получить консультацию ИИ-ассистента, оформить заявку на консультацию со специалистом, а психологам — вести приём заявок и общаться с пользователями через встроенный чат.

---

## Роли пользователей

В системе три роли:

| Роль | Описание |
|------|----------|
| **Пользователь** | Доступ к главному меню, создание заявок, чат с ИИ, общение с психологом |
| **Психолог** | Просмотр очереди заявок, работа с заявками, чат с пользователями, смена статусов |
| **Администратор** | Управление психологами (назначение/снятие), назначение заявок, просмотр всех заявок |

---

## Команды бота

| Команда | Где работает | Описание |
|---------|-------------|----------|
| `/start` | Везде | Приветствие и переход в главное меню (для психолога — панель психолога) |
| `/menu` | Везде | Возврат в главное меню из любого состояния |
| `/clear` | Чат с ИИ | Очистка истории диалога и пользовательской памяти |
| `/help` | Везде | Справка по командам |
| `/end` | Режим чата | Завершение активного чата (работает и у психолога, и у пользователя) |

## Возможности для пользователя

### Главное меню (4 пункта)

1. **Консультация со специалистом** — оформление заявки через пошаговую форму
2. **Консультация с ИИ** — переход в режим чата с ИИ-ассистентом
3. **Условия обращения** — просмотр условий и политики конфиденциальности
4. **Вопрос по психологии** — быстрый вопрос по теме психологии

### Оформление заявки

Пошаговая форма из 5 этапов:

1. Тема обращения (свободный текст)
2. Пол
3. Возраст (от 1 до 120, с проверкой корректности)
4. Степень серьёзности (Лёгкая / Средняя / Высокая / Критическая)
5. Подробное описание проблемы

После заполнения создаётся заявка со статусом «Новое». Пользователь получает подтверждение.

### Чат с ИИ

- Персональный диалог с ИИ-ассистентом (модель настраивается)
- Запоминание контекста разговора (до 20 сообщений)
- Система памяти: автоматическое создание краткой сводки каждые 12 сообщений
- Детекция кризисных ситуаций (суицидальные мысли, самоповреждение) с выводом контактов экстренной помощи
- Защита от спама: ограничение частоты сообщений (не чаще 1 раза в 4 секунды, до 12 в минуту)
- Распознавание приветствий и мелких фраз без обращения к ИИ
- Очистка истории командой `/clear`

### Чат с психологом

- Когда психолог начинает чат по заявке, пользователь получает уведомление
- Все сообщения пользователя пересылаются психологу, и наоборот
- Завершить чат можно командой `/end` или `/menu`
- После завершения пользователь возвращается в главное меню

## Возможности для психолога

### Панель психолога (2 пункта)

1. **Очередь заявок** — все заявки со статусом «Новое» и «Ожидание ответа» (в том числе от других психологов)
2. **Мои заявки** — заявки, назначенные на текущего психолога

Списки поддерживают **пагинацию** — по 10 заявок на страницу с навигацией: `далее`, `назад`, `отмена`.

### Карточка заявки

При выборе заявки из списка открывается карточка с полной информацией:

- Тема, пол, возраст, серьёзность, описание
- Текущий статус и назначенный психолог
- Доступные действия (зависят от состояния заявки):
  - **Взять в работу** — назначить заявку на себя (если ещё не назначена)
  - **Изменить статус** — перевести заявку в один из статусов:
    - В работе
    - Ожидание ответа
    - Закрыто
  - **Начать чат** / **Закрыть чат** — запуск или завершение чата с пользователем
  - **Назад к списку** — возврат к списку заявок

### Чат с пользователем

- Психолог входит в режим чата через карточку заявки
- Все сообщения отображаются с коротким хеш-номером заявки (например, `#A3F2B1`)
- Пользователь автоматически получает уведомление о начале чата
- Завершить чат: `/end`, `/menu`, `/start` или кнопка «Закрыть чат»
- При завершении чата обе стороны получают уведомление
- Сообщения сохраняются в истории заявки (с временными метками)

## Возможности для администратора

### Панель администратора (4 пункта)

1. **Управление психологами**
   - Список всех психологов с количеством активных заявок
   - Добавить психолога — по Telegram ID или @username
   - Снять роль психолога — выбор из пагинированного списка
2. **Все заявки**
   - Последние 10 заявок с индикаторами серьёзности (🔴🟠🟡🟢)
   - Отображение статуса и назначенного психолога
3. **Назначить на заявку** — двухшаговый процесс:
   1. Выбор незакреплённой заявки
   2. Выбор психолога (сортировка по нагрузке — сначала менее загруженные)
4. **Обычное меню** — переход в меню обычного пользователя

## Система заявок

### Статусы заявок

| Статус | Описание |
|--------|----------|
| **Новое** | Только что создана, не назначена |
| **В работе** | Психолог взял в работу |
| **Ожидание ответа** | Ожидание ответа пользователя |
| **Закрыто** | Заявка завершена |

### Уровни серьёзности

| Уровень | Индикатор |
|---------|-----------|
| Критическая | 🔴 |
| Высокая | 🟠 |
| Средняя | 🟡 |
| Лёгкая | 🟢 |

Заявки сортируются по серьёзности (от критической к лёгкой), затем по дате создания.

## ИИ-функции

### Модель

- Провайдер: **OpenRouter** (https://openrouter.ai)
- Рекомендуемая модель: `arcee-ai/trinity-large-preview:free` (бесплатная, настраивается через `OPENROUTER_MODEL` в `.env`)

### Детекция кризиса

Бот распознаёт 27+ ключевых слов и фраз, связанных с суицидом и самоповреждением. При обнаружении кризисной ситуации:

- Выводится специальное сообщение с контактами экстренной помощи
- Телефон доверия: **8-800-2000-122** (бесплатно, круглосуточно)
- Международный ресурс: https://findahelpline.com

### Память пользователя

- Хранит сводку разговора (3–5 предложений) и последние 8 сообщений
- Автоматически обновляет сводку каждые 12 сообщений
- Сводка передаётся ИИ как контекст для более осмысленных ответов
- Сбрасывается командой `/clear`

### Защита от спама

- Минимальный интервал между сообщениями: 4 секунды
- Максимум запросов в минуту: 12
- Распознавание одинаковых повторных сообщений с возвратом кешированного ответа
- Ограничение длины входящих и исходящих сообщений: 1200 символов

## Хранение данных

### База данных — SQLite

| Таблица | Назначение |
|---------|-----------|
| `sessions` | Состояние сессий пользователей (состояние, данные формы, контекст ИИ, пагинация, активный чат) |
| `tickets` | Заявки (тема, данные пользователя, серьёзность, статус, назначенный психолог, история чата) |
| `user_roles` | Профили и роли пользователей (роль, username, имя, дата создания) |

### Память пользователя

Хранится в отдельном JSON-файле (путь задаётся переменной `MEMORY_STORE_PATH`).

## Система уведомлений

Бот использует очередь уведомлений (`pending_notifications`) для кросс-пользовательского общения:

- Уведомление психологу при назначении заявки
- Уведомление пользователю при начале/завершении чата
- Уведомление психологу при получении сообщения от пользователя
- Уведомления отправляются автоматически после обработки каждого входящего сообщения
