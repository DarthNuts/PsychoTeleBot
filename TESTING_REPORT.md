## Найденные и исправленные баги

### 🐛 Баг #1: Автоматическое создание профилей админов
**Проблема:** В `__init__` автоматически создавались профили для всех admin_ids  
**Тест:** `test_list_admins_only_created`  
**Исправление:** Убрана автоматическая инициализация, профили создаются при первом взаимодействии

### 🐛 Баг #2: Обновление метаданных существующих пользователей
**Проблема:** `get_or_create_user` обновлял username/first_name/last_name при каждом вызове  
**Тест:** `test_get_or_create_existing_user`  
**Исправление:** Метаданные обновляются только при создании нового пользователя

### 🐛 Баг #3: is_psychologist возвращал true для админов
**Проблема:** Метод проверял `role in (PSYCHOLOGIST, ADMIN)`  
**Тест:** `test_is_psychologist_admin`  
**Исправление:** Теперь проверяет только `role == PSYCHOLOGIST`

### 🐛 Баг #4: list_psychologists включал админов
**Проблема:** Список включал пользователей с ролью ADMIN  
**Тест:** `test_list_psychologists_excludes_admins`  
**Исправление:** Фильтр изменен на `role == PSYCHOLOGIST`

### 🐛 Баг #5: is_admin не работал без созданного профиля
**Проблема:** Метод проверял только профиль в users  
**Тест:** `test_is_admin_without_profile`  
**Исправление:** Добавлена проверка admin_ids напрямую

### 🐛 Баг #6: String conversion enum
**Проблема:** Ожидалось значение "user", но получали "UserRole.USER"  
**Тест:** `test_user_role_string_conversion`  
**Исправление:** Тест изменен на проверку .value вместо str()
