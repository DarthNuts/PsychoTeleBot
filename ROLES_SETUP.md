# Настройка системы ролей

## Обзор

Бот PsychoTeleBot теперь поддерживает три уровня ролей:

1. **Пользователь** (USER) - по умолчанию для всех
2. **Психолог** (PSYCHOLOGIST) - доступ к заявкам и чату
3. **Администратор** (ADMIN) - управление психологами и полный доступ

## Настройка администраторов

### 1. Получите свой Telegram ID

Отправьте любое сообщение боту [@userinfobot](https://t.me/userinfobot) или используйте [@getmyid_bot](https://t.me/getmyid_bot)

Вы получите свой ID (например: `123456789`)

### 2. Настройте .env файл

Откройте файл `.env` и добавьте ID администраторов:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
ADMIN_IDS=123456789,987654321
```

**Важно:**
- ID указываются через запятую без пробелов
- Можно указать одного или нескольких админов
- После изменения нужно перезапустить бота

### 3. Перезапустите бота

```bash
# Windows
run_telegram.bat

# Linux/Mac
./run_telegram.sh
```

## Использование ролей

### Для администратора

После запуска бота администраторы увидят специальное меню:

```
👑 АДМИН-ПАНЕЛЬ

1️⃣ Управление психологами
2️⃣ Все заявки
3️⃣ Назначить на заявку
4️⃣ Обычное меню
```

#### Добавление психолога

1. Выберите "1" или напишите "управление психологами"
2. Получите Telegram ID пользователя, которого хотите сделать психологом
3. Отправьте этот ID боту
4. Готово! Пользователь теперь психолог

#### Просмотр всех заявок

1. Выберите "2" или напишите "все заявки"
2. Увидите список всех заявок с их статусами

### Для психолога

Психологи видят специальную панель:

```
🧑‍⚕️ ПАНЕЛЬ ПСИХОЛОГА

1️⃣ Очередь заявок
2️⃣ Мои заявки
3️⃣ Обычное меню
```

#### Работа с заявками

- **Очередь заявок** - новые заявки, ожидающие ответа
- **Мои заявки** - заявки, взятые вами в работу

### Для обычного пользователя

Обычные пользователи видят стандартное меню:

```
🏠 Главное меню

1️⃣ Записаться на консультацию
2️⃣ Задать вопрос ИИ
3️⃣ Условия консультации
4️⃣ Задать вопрос психологу
```

## Хранилище данных

Все данные сохраняются в SQLite базу данных `bot_data.db`:

- **Сессии пользователей** - текущее состояние, формы
- **Заявки** - все заявки с историей чата
- **Роли** - роли пользователей и их профили

### Резервное копирование

Рекомендуется регулярно делать бэкап файла `bot_data.db`:

```bash
# Копирование БД
cp bot_data.db bot_data_backup.db
```

## Безопасность

⚠️ **Важные правила:**

1. **Не публикуйте .env файл** - он содержит токен бота и ID админов
2. **Не делитесь ID администраторов** - это доступ к управлению ботом
3. **Регулярно проверяйте список психологов** - удаляйте неактивных
4. **Делайте бэкапы БД** - данные не восстановятся при потере

## Проверка работы

### 1. Проверка администратора

1. Запустите бота
2. Отправьте `/start`
3. Должны увидеть "👑 АДМИН-ПАНЕЛЬ"

### 2. Проверка назначения психолога

1. Создайте второй Telegram аккаунт (или попросите друга)
2. Запустите бота из админ-аккаунта
3. Выберите "Управление психологами"
4. Введите ID второго аккаунта
5. Запустите бота из второго аккаунта
6. Должны увидеть "🧑‍⚕️ ПАНЕЛЬ ПСИХОЛОГА"

### 3. Проверка базы данных

```bash
# Проверить что БД создана
ls -la bot_data.db

# Посмотреть содержимое (требует sqlite3)
sqlite3 bot_data.db "SELECT * FROM user_roles;"
```

## Устранение проблем

### Администратор не видит админ-панель

1. Проверьте что ID в .env правильный
2. Проверьте отсутствие пробелов в ADMIN_IDS
3. Перезапустите бота
4. Отправьте `/start` заново

### Психолог не видит панель психолога

1. Проверьте что администратор добавил его ID
2. Перезапустите бота психологом
3. Отправьте `/start` заново

### База данных не создается

1. Проверьте права на запись в папку
2. Проверьте что нет файла `bot_data.db` с защитой от записи
3. Посмотрите логи бота на ошибки
